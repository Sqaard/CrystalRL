# R6c architecture & control-knob weakness analysis (controllability lens)

**Plan A of the CRYSTAL-1 controllability set.** Focus: *steerability, legibility, authority, reversibility, composition, command-certification* — **not** alpha/returns. Every code/architecture claim is grounded in a real file (`path:line`). External literature is tagged **Tier-A** (named author/method) or **Tier-B** ([class]); evidence carries a status **Established / Strong-but-Contextual / Plausible / Speculative**. Settled facts from `reports/_crystal_ctrl/ANCHORS.md` are not re-litigated: **R6c > CRYSTAL-1 on controllability today**; the near-term substrate is HL-over-R6c; CRYSTAL-1 is the born-legible successor whose K-simplex belief *is* the writable command surface.

## 0. Architecture recap (grounded)

R6c is a **PPO Beta-cash-split + Dirichlet-stock policy wrapped in a deterministic execution controller stack.** The learned net is small: `pi:[128,64]`, `vf:[256,128]` (`configs/stage0_1_active_r_pipeline.yaml:60-67`). The actor MLP's terminal layer is a **64-d penultimate latent** feeding `action_net` — for the root-split family a `Linear(latent_dim, 2 + stock_dim)` that parameterizes a Beta over invested fraction `q` and a Dirichlet over the risky simplex (`src/ppo/dirichlet_policy.py:800-826`, `RootSplitBetaDirichletDistribution`). The two-agent budget variant splits this into `BudgetPMActorCritic` (Beta `q_target` + discrete review horizon; `w1_budget_trader_policy.py:136-213`) over `BudgetTraderActorCritic` (per-stock shared encoder → daily Dirichlet full-portfolio simplex; `:228-426`), with an optional `GraphHierarchicalAssetEncoder` (residual-correlation + group-pool message passing; `:36-111`). **The policy's learned output is deliberately thin: a cash stance and a near-equal-weight risky tilt.** Almost all *named, legible* behavior lives in the **deterministic K-window execution controller** downstream of the sampled action: PD schedule → mild root confidence slice → incremental Top-K flow → group-aware caps → risk-aware buy-gate/sell-priority → stock confidence slice → executed weights (`reports/r_k_window_analysis/R6C_GROUP_RISKAWARE_TOPK_IMPLEMENTATION_NOTE.md:19-46`). This is the whole controllability paradox of R6c: **the legible part is a hand-written controller (~214 config knobs), and the learned part is an unnamed 64-d vector.** The R6c+ latent-action head (`stage0_1_weight_env.py:2879-2912`) is the one place a *native discrete named primitive* was retrofitted, but on Dow-29 its prototypes' risky part is ~equal-weight — "no stock-selection language" — so even there the only real signal it commands is the cash stance (`:2882-2884`).

## 2. The 3–5 highest-leverage fixes (ranked), and the migrate-to-CRYSTAL-1 argument

Ranked by **controllability leverage per unit blast-radius**, not by return.

**Fix 1 — Replace the unnamed 64-d steering surface with a born-named write (W1+W2+W5).**
This is one fix because W1 (unnamed latent), W2 (implicit belief), and W5 (off-manifold forcing) are the same root cause: *there is no factored, named, on-manifold state to write to.* The K-means codebook is a read-only C0 layer at best; the crisp-primitives file already proved the latent is continuous (`r6c_crisp_primitives_summary.json:49`), so a "native code" on R6c is only reachable by a **T2 VQ-bottleneck retrain**. **This is the single strongest argument FOR migrating the mechanism into CRYSTAL-1**, whose `NeuralBayesFilter` K-simplex belief *is* the born-legible, on-manifold, scalar-writable command surface (`src/crystal/belief_filter.py`; `ANCHORS:34-40`). Verdict: **migrate — do not retrofit R6c.** Cross-ref: HL5-IV C0 addressability + C1 causal-patch ladder (`IV:31-32`).

**Fix 2 — Instantiate the HL5-K registry on R6c: ~214 → ~20 typed levers, the rest fenced/merged (W3+W8).**
Purely config-side (T0), immediately deployable, and it is the *near-term* controllability win because R6c is the near-term substrate (`ANCHORS:4-5`). The disposition is already specified (`HL5..._K:446-473`): MERGE 30 confidence keys→3 intent levers, 7 group coeffs→1 hard cap; FENCE reward/PPO/estimator internals; EXPOSE ~18–20 levers each carrying the typed tuple (`:287-319`). This does **not** argue for migration — it is the mechanism HL-over-R6c ships *and* the type system CRYSTAL-1 inherits. It is genuinely shared IP; reuse HL5-K verbatim. **Argue: keep on R6c, port the grammar to C1.**

**Fix 3 — Wire the IV C6 certificate + cumulative-`D` ledger into the live HCS path (W6+W11).**
Today R6c has forensic pieces (ghost ledger, deflated Sharpe, OODGate) but no *live* writ object and no cumulative-authority budget, so salami-slicing and read→write certificate reuse are unguarded (`IV:32,285`). Wiring the single-writ schema (`IV:150-226`) + a per-principal signed `D` ledger (`IV:281`) + co-activation cap `N_live≤K` (the blunt load-bearing limiter while `D_total` is uncalibrated, `IV:327`) is T0/T1 and applies to *both* agents. This argues **weakly for migration**: the ledger is agent-agnostic, but a certificate is far cheaper to write and bound on C1's scalar named write than on R6c's entangled latent — so the *same* governance is more affordable on C1. Cross-ref the IV writ-ladder C0–C6 directly.

**Fix 4 — Make every mode-flip / law-reversion an annunciated GUARANTEE-DELTA (W7), and name `correction_penalty`'s `Tt` (W4).**
Two related T0 fixes on the authority axis. R6c's risk-aware gates and overlay silently change the live guarantee set; ship the MEL degraded-knob table (`HL5..._K:360-402`) so each demotion emits `{guarantees_now_void[], hazard_rank}`, latched + ack-required (`IV:186-192`). In the same pass, promote `correction_penalty` from an implicit tax to a first-class anti-windup channel with a named `Tt`, alarming on the gap-integral and propagating saturation to the HL update layer so windup cannot migrate into rule coefficients (`IV:108`). Both are config/logging-side. This is **neutral on migration** — it is hygiene R6c needs to be governable at all; C1 gets the same machinery but starts with a legible mode structure.

**Fix 5 (contingent) — VQ-bottleneck retrain OR native selection primitives (W10, the T2 residual).**
Only if a *native* named action space on the real universe is required and the C1 migration is deferred. This is the expensive end (T2 architecture): a discrete VQ bottleneck in the policy would give R6c born-crisp codes (`r6c_crisp_primitives_summary.json:49`), and a richer universe than Dow-29 would give the latent-action head real stock-selection language (`stage0_1_weight_env.py:2882`). It is listed last because it **duplicates on R6c exactly what CRYSTAL-1 was built to be** — so it is the clearest "don't do this, migrate instead" line. Keep `primitive_regime_prior` (`knob_registry.yaml:54`) as the cheap partial win in the meantime.

**One-line ranking rationale:** Fixes 2 & 4 are T0 and ship now on R6c (near-term substrate). Fix 3 is the shared governance spine. Fixes 1 & 5 are the T2 fork — and both resolve to *migrate the mechanism into CRYSTAL-1's born-legible design* rather than retrofit an unnamed latent.
