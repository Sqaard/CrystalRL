CrystalRL — live testing

An interactive sandbox for one lesson: different models respond differently to the same knobs, and different knobs have different effects. Tick a model on/off, move a knob, watch the curves — everything recomputes instantly on a seeded synthetic market.

R6c — the CHRL-line agent (latent core)

How it works: a PPO network reads the market panel into a 64-dimensional unnamed latent; three readable control layers (risk → groups → stocks) plus guardrails (exposure pacing, confidence scaling, a buy gate, event de-risk) shape the final weights. Transparent layers, opaque core: you steer it with engineering knobs, and there is no named memory you can write a command to.

CRYSTAL-1 — legible by construction

How it works: the only memory is a named belief — P(bear) — updated by a Bayes filter; the policy is a small readable rule over that belief (in the real system: an ≤8-leaf soft tree at full return parity). Because the memory is named, SET_BELIEF is a command the policy must obey — and because the objective is certified, some interventions get refused.

C-1 named commands (the crystal agent listens)

The killer demo: crank SET_BELIEF write strength to 100%. CRYSTAL-1 obeys the written belief (it is a command). R6c cannot even receive it — there is no named memory to write to. That asymmetry is the whole thesis.
